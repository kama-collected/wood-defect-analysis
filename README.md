# Wood Defect Detection — DIP Assignment

1. **Augmentation & Splitting** (`augment.ipynb`) — train/val/test split (70 / 15 / 15 %) with 6-variant augmentation applied to the training set, producing a 7× multiplier.
2. **Pre-processing** (`pre-processing.ipynb`) — noise reduction, contrast enhancement, sharpening, edge overlay, and colour normalisation.


## Dataset

| Property | Value |
|---|---|
| Source | Roboflow — `digital-image-processing-utar` |
| Images | 4 177 training images |
| Format | YOLO (bounding box `.txt` labels) |
| Classes | Crack, Dead\_Knot, Healthy, Knot\_missing, Knot\_with\_crack, Live\_Knot, Marrow, Quartzity, Resin |

- Raw images live in `dataset/train/images/`. Corresponding labels are in `dataset/train/labels/`.
- Augmented outputs are written to `dataset/split/`.
- Preprocessed outputs are written to `dataset/preprocessed/`.
   - To change the sample image, replace the value for `SAMPLE_IDX` in Cell 5 of `pre-processing.ipynb`.
   - You can delete the augmented images once you achieve the desired preprocessed images to save some space on your computer


### 3 — Install dependencies

```bash
pip install -r requirements.txt
```

---

### 4 — Select the virtual environment kernel in VS Code (Jupyter)

1. Open `yolo26/pre-processing.ipynb` in VS Code.
2. Click the kernel selector in the top-right corner of the notebook (it may show **"Select Kernel"** or a Python version).
3. Choose **"Python Environments…"** → select the `.venv` interpreter (it will be listed as `.venv`).
4. VS Code will use this kernel for all notebook cells.

> **Tip:** If `.venv` does not appear, press `Ctrl+Shift+P`, run **"Python: Select Interpreter"**, and point it to `.venv\Scripts\python.exe` (Windows) or `.venv/bin/python` (macOS/Linux). Then reopen the kernel picker.

---

### 5 — Run the augmentation notebook

Open `yolo26/augment.ipynb` and run **all cells in order**.

The notebook will:

| Step | Cell | Description |
|---|---|---|
| 1 | Setup | Imports, path config, split ratios (70 / 15 / 15 %), class names |
| 2 | Helpers | Define `load_image`, `load_labels`, `save_image`, `save_labels`, `draw_bboxes` |
| 3 | Augmentation functions | `rotate_yolo_labels` (affine bbox transform) and `augment_image` (6 variants) |
| 4 | Split dataset | Shuffle with `RANDOM_SEED=42`, partition into train / val / test |
| 5 | Augment training split | Write 7 files per training image (orig + hflip, vflip, rot +15°, rot −15°, bright, dim) |
| 6 | Copy val & test splits | Copy validation and test images/labels without augmentation |
| 7 | Write YAML | Generate `dataset/data_split.yaml` with correct split paths |
| 8 | Verification | Print split summary, assert no augmented files leaked into val/test, show 4 random pairs |

Augmented outputs are written to `dataset/split/`. The 7× multiplier means a 4 177-image training set grows to ~20 000 training files.

---

### 6 — Run the pre-processing notebook

Open `yolo26/pre-processing.ipynb` and run **all cells in order** (`Run All` in the toolbar, or `Ctrl+Shift+P` → **"Notebook: Run All Cells"**).

The notebook will:

| Step | Cell | Description |
|---|---|---|
| 1 | Setup | Import libraries |
| 2 | Config | Set input/output paths and class names |
| 3 | Helpers | Define `load_image`, `load_labels`, `draw_bboxes`, etc. |
| 4 | Exploration | Count images/labels, show class distribution bar chart |
| 5 | Histogram analysis | Sample 50 images, plot brightness histograms, suggest enhancement strategy |
| 6 | Gaussian blur | Noise reduction (kernel 3×3, σ=0.85) |
| 7 | Histogram equalisation | Y-channel equalisation in YCrCb space |
| 8 | CLAHE | Adaptive contrast (clip=2.8, tile 6×6) on LAB L-channel |
| 9 | Sharpening | Unsharp masking (σ=1.0, α=1.5) |
| 10 | Edge overlay | Canny edge blend (t1=50, t2=150, α=0.4) |
| 11–12 | Colour normalisation | LAB normalisation and batch preprocessing |

Preprocessed images and a new `data_preprocessed.yaml` are saved under `dataset/train/preprocessed/`.

## Augmentation Pipeline

Six augmented variants are generated for every training image, giving a **7× dataset multiplier**:

| Variant | Tag | Transform |
|---|---|---|
| Original | `orig` | No change — raw image copied as-is |
| Horizontal flip | `hflip` | `cv2.flip(img, 1)` — bbox x-centres mirrored |
| Vertical flip | `vflip` | `cv2.flip(img, 0)` — bbox y-centres mirrored |
| Rotation +15° | `rot_pos` | Affine rotation; bbox corners transformed and re-fitted |
| Rotation −15° | `rot_neg` | Affine rotation; bbox corners transformed and re-fitted |
| Brightness boost | `bright` | `alpha=1.2, beta=+30` — labels unchanged |
| Brightness dim | `dim` | `alpha=0.85, beta=−20` — labels unchanged |

Val and test splits are **never augmented** — only raw images are copied, ensuring unbiased evaluation.

## Pre-processing Pipeline

```
Raw image
   │
   ▼
Gaussian Blur  (noise reduction, kernel=3, σ=0.85)
   │
   ▼
CLAHE  (adaptive contrast, clip=2.8, tile=6×6)
   │
   ▼
Sharpening / Unsharp Mask  (σ=1.0, α=1.5)
   │
   ▼
Edge Overlay  (Canny blend, optional — beneficial for Crack class)
   │
   ▼
LAB Colour Normalisation  (optional)
   │
   ▼
Preprocessed image  →  dataset/train/preprocessed/
```

Each technique can be toggled via the boolean flags at the top of the batch cell (`USE_CLAHE`, `USE_GAUSSIAN_BLUR`, `USE_SHARPENING`, `USE_EDGE_OVERLAY`, `USE_COLOR_NORM`).
